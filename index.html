Currently Building

A graph-native schema registry where:

  • Every schema version is a node with column-level features (types, nullability, cardinality).
  • Every transformation between versions is a weighted directed edge (additive = low weight, breaking = high weight).
  • A GNN operates on this graph to predict breaking downstream consumers, safest migration paths, and drift trajectories.
  • SQL migrations are auto-generated by traversing the shortest path through the transformation graph.
